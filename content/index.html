About WolGoWeb

WolGoWeb is a remote Wake-on-LAN WebAPI tool developed with Golang, designed for deployment on local network servers or NAS devices to wake up hosts on the local network via WebAPI. The project has been tested for 5 years and runs stably in many testing and production environments.

ğŸ’¡ Why Choose WolGoWeb?
  • No complex configuration, one-click deployment
  • Supports both Web interface and API
  • Optional authentication mechanism for security
  • Cross-platform support for various environments
  • Deploy with one click on Lazycat platform without worrying about container configuration

Main Features

  • WebAPI Wake: Send WOL magic packets through simple HTTP interface
  • Lightweight Design: No database required, call directly via API
  • API Key Verification: Supports API key verification mechanism (optional)
  • Multi-device Support: Use different MAC addresses for different devices
  • Custom Configuration: Supports custom ports, broadcast addresses, network interfaces, etc.
  • Cross-platform Support: Supports Windows, Linux, FreeBSD, etc.

API Usage

Basic Wake

Wake up devices with a simple GET request:

Browser Access: https://wolgoweb.your-box.heiyu.space/wol?mac=AA-BB-CC-DD-EE-FF
Command Line (curl): 
curl "https://wolgoweb.your-box.heiyu.space/wol?mac=AA-BB-CC-DD-EE-FF"

Advanced Parameters

Parameter Description Default
mac Target device MAC address (required) -
ip Broadcast address 255.255.255.255
port WOL port 9
network Network interface name Default interface
key API key (required when authentication is enabled) -

Custom Broadcast Address and Port

If API key verification is enabled, you need to add the key parameter in the request:

Using crontab scheduled tasks, you can set up regular device wake-ups, suitable for remote work or laboratory scenarios.

Example: Automatically wake up computer at 8:00 AM every day

More Examples:

  • Wake up at 7:30 AM Monday to Friday: 30 7 * * 1-5 curl "https://..."
  • Wake up every hour: 0 * * * * curl "https://..."
  • Wake up at 11:00 PM daily: 0 23 * * * curl "https://..."

Scenario 3: Automation Integration

Through the open API interface, it can be integrated with other automation tools (such as Home Assistant, Node-RED) to achieve more complex smart home scenarios.

Scenario 4: Remote Work

Access the web interface through your mobile browser when you're out, remotely wake up your computer at home or in the office, and start working anytime.

FAQ

Q: Why does the app interface show http:// addresses?

A: The original WolGoWeb application displays example addresses using the http:// protocol by default in its web interface. However, on the Lazycat platform, all applications are accessed via HTTPS encryption.

Solution: When actually using the service, replace all http:// shown in the app interface with https://. For example:

  • Interface shows: http://wolgoweb.your-box.heiyu.space/wol?mac=...
  • Actual use: https://wolgoweb.your-box.heiyu.space/wol?mac=...

Q: Why can't the device wake up?

A: Please check the following:

  • Ensure Wake-on-LAN is enabled in the target device's BIOS/UEFI
  • WOL function is enabled in network card driver settings
  • MAC address is entered correctly (supports AA:BB:CC:DD:EE:FF or AA-BB-CC-DD-EE-FF format)
  • Device and WolGoWeb are on the same local network
  • Some routers may require port forwarding configuration or allowing broadcast packets

Q: How to get the device's MAC address?

A: Different methods for different systems:

  • Windows: Open Command Prompt, run ipconfig /all
  • macOS: System Preferences â†’ Network â†’ Advanced â†’ Hardware
  • Linux: Run ip link show or ifconfig

Q: Does it support cross-subnet wake-up?

A: The WOL protocol itself is based on local area network broadcasting. Cross-subnet wake-up requires router support for directed broadcast. It is recommended to use on the same local network.

Q: How to modify environment variable configuration?

A: On the application details page of the Lazycat platform, find the "Environment Variables" configuration item, modify it, save and restart the application to take effect.
